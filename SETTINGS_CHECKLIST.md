# ✅ Settings Database - قائمة التفقد الكاملة

## 🎯 قبل البدء

- [ ] لديك حساب Supabase
- [ ] فتحت مشروعك في Supabase
- [ ] لديك صلاحيات Admin
- [ ] متصفح محدّث

---

## 📋 الخطوة 1: إنشاء الجدول (10-15 دقيقة)

### 1.1 افتح Supabase
- [ ] اذهب إلى https://app.supabase.com
- [ ] اختر مشروعك
- [ ] من القائمة اليسار، اضغط **SQL Editor**

### 1.2 أنشئ Query جديدة
- [ ] اضغط **New Query** أو **+**
- [ ] يجب أن ترى محرر SQL فارغ

### 1.3 اختر وانسخ الـ Script
**الخيار A - للمبتدئين (موصى به):**
- [ ] افتح: `SUPABASE_SETTINGS_TABLE.sql`
- [ ] انسخ كل المحتوى (Ctrl+A ثم Ctrl+C)
- [ ] الصق في SQL Editor (Ctrl+V)

**الخيار B - للمتقدمين:**
- [ ] افتح: `SUPABASE_SETTINGS_COMPLETE.sql`
- [ ] انسخ كل المحتوى (Ctrl+A ثم Ctrl+C)
- [ ] الصق في SQL Editor (Ctrl+V)

### 1.4 تنفيذ الـ Script
- [ ] اضغط **Run** أو اضغط **Ctrl+Enter**
- [ ] انتظر حتى ينتهي التنفيذ (قد يستغرق ثانية أو اثنين)

### 1.5 تحقق من النتائج
يجب أن تظهر نتيجتان في الأسفل:

**النتيجة 1:** عدد الإعدادات
```
settings_count
1
```
- [ ] ظهرت النتيجة الأولى بقيمة 1

**النتيجة 2:** بيانات الإعدادات
```
restaurant_name | phone     | email                   | address
"مطعم بزوم"    | "012345..." | "bazzomrestaurant..." | "دمياط , ميدان..."
```
- [ ] ظهرت البيانات بشكل صحيح

### 1.6 تحقق من الجدول
في Supabase:
- [ ] اذهب إلى **Table Editor**
- [ ] يجب أن ترى جدول جديد اسمه `settings`
- [ ] انقر عليه
- [ ] يجب أن ترى صف واحد مع البيانات

---

## 💻 الخطوة 2: اختبر من التطبيق (5 دقائق)

### 2.1 افتح المتصفح
- [ ] اذهب إلى `http://localhost:3000/admin/settings`
- [ ] إذا لم تكن logged in، سجل دخولك

### 2.2 تحقق من الحقول
يجب أن ترى جميع الحقول التالية:

**إعدادات المطعم:**
- [ ] اسم المطعم
- [ ] رقم الهاتف
- [ ] رقم الواتس (حقل جديد!)
- [ ] البريد الإلكتروني
- [ ] العنوان
- [ ] وقت الفتح
- [ ] وقت الإغلاق
- [ ] رسوم التوصيل
- [ ] الحد الأدنى للطلب
- [ ] وصف المطعم

**إعدادات التطبيق:**
- [ ] وضع الصيانة (toggle)
- [ ] تفعيل الدفع الإلكتروني (toggle)
- [ ] تفعيل الدفع النقدي (toggle)
- [ ] تفعيل التوصيل (toggle)
- [ ] تفعيل الطلب في المطعم (toggle)
- [ ] الحد الأقصى للطلبات يومياً
- [ ] متوسط وقت التوصيل

### 2.3 اختبر Toggles
- [ ] اضغط على واحدة من الـ toggles
- [ ] يجب أن تتغير اللون من أبيض إلى ذهبي (أو العكس)
- [ ] اضغط مرة أخرى للتأكد

### 2.4 عدّل بيانات
- [ ] غيّر رقم الهاتف (مثلاً: 01007576444)
- [ ] غيّر الواتس (مثلاً: 01007576444)
- [ ] تأكد أن الحقول تقبل الإدخال

### 2.5 احفظ الإعدادات
- [ ] اضغط زر **حفظ الإعدادات** (اللون الذهبي)
- [ ] يجب أن تظهر رسالة نجاح بـ toast
- [ ] النص يجب أن يقول "تم حفظ الإعدادات بنجاح!"

### 2.6 تحقق من قاعدة البيانات
- [ ] اذهب إلى Supabase Dashboard
- [ ] يذهب إلى **Table Editor**
- [ ] اضغط على جدول `settings`
- [ ] تحقق من أن البيانات تحديثت (الهاتف والواتس يجب أن تكونوا مختلفة)

## 📊 الخطوة 3: اختبر القراءة (3 دقائق)

### 3.1 افتح Home Page
- [ ] اذهب إلى `http://localhost:3000`
- [ ] تحميل الصفحة بنجاح

### 3.2 تحقق من البيانات
- [ ] في الأسفل، يجب أن ترى قسم **تواصل معنا**
- [ ] البيانات المعروضة يجب أن تكون من Supabase
- [ ] إذا غيرت رقم الهاتف، يجب أن يظهر الرقم الجديد

### 3.3 اختبر القراءة مباشرة (اختياري)
فتح Developer Tools (F12) واكتب:
```javascript
const settings = await fetch('/api/settings').then(r => r.json());
console.log(settings);
```
- [ ] يجب أن ترى البيانات من Supabase

## 🔒 الخطوة 4: التحقق الأمني (2 دقيقة)

### 4.1 تحقق من RLS
في Supabase:
- [ ] اذهب إلى **SQL Editor**
- [ ] شغل:
```sql
SELECT * FROM public.settings;
```
- [ ] يجب أن ترى البيانات بنجاح

### 4.2 تحقق من Policies
في Supabase:
- [ ] اذهب إلى **Authentication** > **Policies**
- [ ] يجب أن ترى:
  - [ ] "Enable read access for all users"
  - [ ] "Enable update for authenticated users only"
  - [ ] "Enable insert for authenticated users only"

## 🐛 استكشاف المشاكل

### ❌ Problem: "Table does not exist"
**الحل:**
- [ ] تأكد من تشغيل SQL Script بالكامل
- [ ] اضغط Run مرة أخرى
- [ ] انتظر حتى ينتهي التنفيذ

### ❌ Problem: "PGRST116"
**الحل:**
- [ ] هذا يعني أنه لا توجد بيانات
- [ ] تأكد من أن الـ INSERT عملت
- [ ] شغل:
```sql
INSERT INTO public.settings (restaurant, app)
VALUES (
  '{"name": "مطعم بزوم", ...}',
  '{"maintenanceMode": false, ...}'
);
```

### ❌ Problem: الحفظ لا يعمل
**الحل:**
- [ ] تحقق من Console (F12)
- [ ] يجب أن تأخذ خطأ واضح
- [ ] تأكد من أنك logged in كـ Admin

### ❌ Problem: البيانات لا تتحدث
**الحل:**
- [ ] افتح Table Editor في Supabase
- [ ] انقر على جدول `settings`
- [ ] تحقق يدوياً من البيانات
- [ ] جرب تعديل يدوي مباشرة
